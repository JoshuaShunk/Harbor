# Governance

This document describes how the Harbor project is governed and how to participate.

## Project Roles

### Users

Anyone using Harbor. Users can:

- Use Harbor freely under the MIT license
- Report bugs and request features via [GitHub Issues](https://github.com/JoshuaShunk/Harbor/issues)
- Ask questions in [GitHub Discussions](https://github.com/JoshuaShunk/Harbor/discussions)
- Submit pull requests

### Contributors

Anyone who has had a pull request merged. Contributors can:

- Everything users can do
- Be credited in release notes
- Participate in design discussions

### Maintainers

Trusted contributors with write access to the repository. Maintainers can:

- Everything contributors can do
- Merge pull requests
- Manage issues and labels
- Cut releases
- Set project direction

Current maintainers:

- [@JoshuaShunk](https://github.com/JoshuaShunk) — Project Lead

### Becoming a Maintainer

Maintainer status is granted by existing maintainers to contributors who have:

- Made multiple quality contributions over time
- Shown understanding of the codebase and project goals
- Demonstrated good judgment in code review
- Been responsive and collaborative

If you're interested, continue contributing and engage in discussions. Maintainers will reach out when appropriate.

## Decision Making

Most decisions are made through discussion in issues and pull requests. For larger changes:

1. **Proposal** — Open an issue or discussion describing the change
2. **Discussion** — Community feedback and iteration
3. **Consensus** — Maintainers seek rough agreement
4. **Implementation** — Code, review, and merge

Maintainers have final say on technical decisions, but strive to incorporate community feedback.

## Pull Request Process

1. Fork the repository and create a branch
2. Make your changes following the [Contributing Guide](CONTRIBUTING.md)
3. Open a pull request with a clear description
4. Address review feedback
5. A maintainer will merge once approved

Pull requests require at least one maintainer approval before merging.

## Code of Conduct

We are committed to providing a welcoming and inclusive environment. Please be respectful in all interactions. Harassment or abusive behavior will not be tolerated.

## Changes to Governance

This governance model may evolve as the project grows. Changes will be proposed via pull request and discussed openly.
